# User Flow Documentation

## Spotify Tracks
- Rate (1-10, public/private)
- Comment
- Tag (pre-made + custom)
- Favorite

## Spotify Albums
- Rate (1-10, public/private)
- Comment
- Tag

## Spotify Artists
- Rate (1-10, public/private)
- Comment
- Tag

## User Playlists
- Rate (1-10, public/private) — *Other users rate your playlists!*
- Comment — *Other users comment on your playlists!*
- Tag
- Contains tracks (each track has its own separate ratings/comments/favorites)

---

# Example User Journey

## Scenario: User "Alice" Creates a Playlist
1. **Alice creates** a playlist named **"Summer Vibes 2024"**.
2. Adds **20 tracks** to it.
3. Tags it with: `summer`, `beach`, `chill`.
4. Marks the playlist as **public**.

---

## Community Interaction
### Bob discovers Alice's playlist in the Community section:
- Rates it: **8/10**
- Comments: *"Great selection! Love track #5"*
- Adds his own tag: `workout-friendly`

### Track-Level Discovery
Track #5, **"Sunset Dreams"**, inside Alice's playlist:
- **142 ratings** (avg **7.8/10**)
- **58 comments**
- Tagged with: `indie`, `summer`, `2024`
- **Favorited by 203 users**

---

#  Community Discovery Features
Users can browse:
- **Most rated playlists** (this week)
- **Top rated tracks** (all-time)
- **Most commented albums** (this month)

And filter by:
- Tags
- Genres
- Rating thresholds
